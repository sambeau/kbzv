# UI feedback #3

Here’s the third round of feedback of the App UI. 

Most of the major and medium issues have been fixed, so lets start honing on on the smaller-sized issues.

There’s still lots of clickable things that don’t have `cursor:pointer`. Is this a Radix issue? While macOS itself uses the Arrow pointer for widgets, once in Safari the standard is to put `cursor:pointer` onto anything clickable. This is my opinion too—`cursor:pointer`is much clearer when the UI is ‘flat’.

[Apple guidelines for Pointers in macOS](https://support.apple.com/en-gb/guide/mac-help/mh35695/mac)

I think it’s heightened by Radix’s subtle mouseover which really benefits from a pointer cursor.
