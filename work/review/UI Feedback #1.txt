## Other important issues 

### Menu

The File menu has one item “Close Window”.

There should be two at the moment.
1. “Open…”
2. “Close”

#### “Open…”

This should open the dialog box to load a project folder.

#### “Close”

This should close the window, but not quit. The user should still be able to open a new window (or manually quit).

The close button on the window should mimic this behaviour.

### Main layout

Main view selector should be `<Tabs>` with three tabs:
- Docs
- Work
- Bugs

### Workflows

#### Can we rename the view “Work”
- Icon is the same as Git, we should change to `list-checks`

#### Two columns should be resizable

The line between should have a resized on mouseover and drag to resize.

#### Workflow list shows “Incidents”, “Decisions”, “Bugs”

1. I don’t know what “Incidents” are. Please remove.
2. “Decisions” are an internal thing, remove for now as humans care about documents.
3. We should move bugs to be a separate list, as there don’t really fit in this hierarchy

#### Filters show “Incident” and “Checkpoint”

I don’t know what these are. Are these entities? I’ve never heard of them. If they are internal we shouldn’t show them. If they don’t exist we need to find out where they came from, e.g. if they’re in the documentation we should remove them.

#### Selecting a filter
- filters should be standard checkboxes
	- When you select a filter the bar grows vertically causing everything to jump
	- When selected a new line of text appears with the colour name with an x next to it. Why not just use a toggle?

#### Colour filters

These are just ‘odd’.
- They are just small coloured circles
- They have no labels
- On mouseover text appears but has no background so it’s hard to understand as it has no background.
- I think we should get rid the color filters for now; keep the colour dots in the list for now
